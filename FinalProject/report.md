# 角色动画控制项目报告

## 1. 项目概述

本项目实现了一个基于 Motion Graph（即项目要求的方案一） 的角色控制器。系统支持键盘或手柄输入，根据用户的速度、转向和动作触发信号，在多段 BVH 动作之间进行切换与混合，实现较自然的待机、行走、奔跑、转向、跳跃以及特殊动作控制。

## 2. 提交文件与使用方式

本次主要修改了以下文件：

- `answer_project.py`
- `viewer/controller.py`

并添加了数据集：
- `excep_motion/dance.bvh`
- `excep_motion/kick.bvh`
- `excep_motion/backflip.bvh`

运行方式与原项目一致：

```bash
python task1_project.py
```

如果检测到手柄，则优先使用手柄输入；检测到手柄后，键盘输入会被忽略。

## 3. 动作素材与状态设计

本项目使用了 `motion_material/` 中提供的多段 BVH 动作，主要包括：

- `idle.bvh`
- `walk.bvh`
- `run.bvh`
- `jump.bvh`
- `turn_left.bvh`
- `turn_right.bvh`
- `excep_motion/dance.bvh`
- `excep_motion/kick.bvh`
- `excep_motion/backflip.bvh`

其中：

- `idle / walk / run` 作为可循环的基础 locomotion 动作
- `jump` 作为一次性动作
- `turn_left / turn_right` 作为转向视觉增强动作
- `dance / kick / backflip` 作为特殊动作，取自[SFU-dataset](https://mocap.cs.sfu.ca/)，kick和backflip是从martial_art中截取的片段

在 `graph.py` 中构建了一个手工设计的 demo motion graph，将上述动作组织成若干节点，并为常见状态切换建立了边。

## 4. 控制方式

### 键盘

- `WASD` 或方向键：控制平面移动方向
- `Shift`：切换到跑步 gait
- `Space`：触发跳跃
- `1`：触发整身 `dance`
- `2`：触发 `kick`
- `3`：触发 `backflip`
- `4`：触发上半身 `dance overlay`

### 手柄

- 左摇杆：控制移动方向
- `RB`：切换跑步 gait
- `B`：触发跳跃
- `Y`：触发整身 `dance`
- `A`：触发 `kick`
- `X`：触发 `backflip`
- `LT + Y`：触发上半身 `dance overlay`

特殊动作可以通过再次动方向键或重新拨动左摇杆打断

## 5. 核心实现方法

### 5.1 Motion Graph 与状态切换

角色当前动作由 `CharacterController` 维护。系统每帧根据用户输入和当前状态，从 motion graph 中选择目标动作，并在合法边内进行切换。

切换流程为：
1. 根据输入速度和 gait 选择目标 locomotion 状态。
2. 如果存在跳跃或特殊动作请求，则优先切换到对应动作。
3. 根据 motion graph 检查当前状态是否允许切换到目标状态。
4. 若允许，则构造 transition，并在若干帧内完成 blend。

### 5.2 动作预处理

在 `answer_project.py` 中对原始动作进行了预处理：

- 对 `walk` 和 `run` 使用 `build_loop_motion()` 构造更平滑的循环动作
- 对 `jump` 只截取起跳到落地的有效区间
- 对 `dance / kick / backflip` 做骨骼重定向，使其映射到项目角色骨架

特殊动作的重定向过程中，使用 `idle` 的骨架作为模板，并通过手工关节映射表将外部骨架关节映射到当前角色骨骼。

### 5.3 基于特征的切换帧选择

为了减少动作切换的跳变，系统在切换时不会总是从目标动作的第 0 帧开始，而是利用以下特征搜索更合适的切入帧：

- 相对关节位置
- 角色朝向
- root 平面速度

对每个候选帧计算代价后，选择代价最小的切换帧。对于 `walk` 和 `run` 之间的切换，还使用了相位邻域搜索，使步态切换更稳定。

### 5.4 Root 对齐与连续控制

动作在切换或重采样前，会先对齐到当前世界空间中的 root 位置和朝向：

- 对齐 root 的平面位置
- 对齐角色朝向
- 将动作整体旋转到当前控制朝向

这样做可以避免不同动作片段在世界坐标中的直接拼接带来明显跳跃。

同时，角色 root 的朝向和速度采用平滑阻尼更新，使输入变化不会造成朝向瞬变。

### 5.5 转向视觉增强

在转向实现过程中，我没有直接将 `turn_left` 和 `turn_right` 当作完整的主动作来驱动角色，这是因为这种做法虽然能够表现“转身”本身，但会把角色朝向变化限制在已有动作片段的形态上，使转向过程显得离散，而且在处理非 90 度这类任意角度转向时往往不够自然。基于这个问题，最终采用了分层处理的思路：角色的实际朝向变化仍由连续控制逻辑负责，而 `turn_left` 和 `turn_right` 只作为视觉增强层混合到当前 locomotion 动作中。这样做的好处是，一方面保留了转向控制的连续性和精确性，另一方面也让角色在转弯时不仅仅是 root 朝向发生变化，而是能在上半身呈现更明显、更自然的转身动作。

具体做法是：

- 根据当前目标朝向与实际朝向的误差计算一个转向权重
- 当误差足够大时，采样转向动作
- 只对上半身相关骨骼进行混合

这样可以在不破坏下半身 locomotion 稳定性的前提下，让转向表现更明显。

## 6. 上下半身分开控制的实现

本项目实现了 README 中的一个加分点：将角色上半身和下半身分开控制，使角色可以组合新动作。

### 6.1 设计思路

下半身继续执行原本的 locomotion 动作，例如：

- `idle`
- `walk`
- `run`
- `turn`

上半身则可以额外叠加一个独立动作层。当前实现的是：

- `LT + Y` 或键盘 `4`：触发上半身 `dance`

触发后，角色下半身仍然保持原来的移动或转向状态，上半身播放 `dance.bvh` 中的动作，从而形成新的组合动作。

### 6.2 骨骼分层方法

系统构建了一个 `upper_body_mask`，按骨骼名称为不同关节赋予不同权重：

- root：0
- pelvis / hip：较小权重
- spine：中等权重
- chest / neck / head：较大权重
- shoulder / arm：适中权重
- leg / foot / toe：0

混合时保留下半身主动作作为 base pose，再对上半身关节使用线性插值和平滑球面插值：

- 平移使用线性插值
- 旋转使用四元数 slerp

同时显式保留 base pose 的 root 位姿，避免上半身动作偷偷影响角色位移。

### 6.3 触发逻辑

为了不破坏原有整身 `dance` 的逻辑，系统将其拆成两种输入：

- `Y`：整身 `dance`
- `LT + Y`：上半身 `dance`

也就是说，同一段 `dance.bvh` 既可以作为整身特殊动作，也可以作为上半身 overlay 动作复用。

本项目当前已实现：
